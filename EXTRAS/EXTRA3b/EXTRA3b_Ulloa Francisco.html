Qu es el modulo Logging?

El mdulo logging de Python es una biblioteca estndar integrada para registrar eventos, errores y mensajes informativos durante la ejecucin de una aplicacin. Permite rastrear el comportamiento del programa, facilitando la depuracin (debugging) y auditora mediante la creacin de archivos de log personalizados con diferentes niveles de gravedad.

Los niveles incluyen:

  • NONSET (nivel 0): Cuando se establece en un logger, indica que los loggers antecesores deben ser consultados para determinar el nivel efectivo. Si el resultado sigue siendo NOTSET, se registrarn todos los eventos.

  • DEBUG (nivel 10): Informacin detallada, normalmente slo de inters para un desarrollador que intenta diagnosticar un problema.

  • INFO (nivel 20): Confirmacin de que todo funciona segn lo previsto.

  • WARNING (nivel 30): Una indicacin de que ha ocurrido algo inesperado o de que podra producirse un problema en un futuro prximo.

  • ERROR (nivel 40): Debido a un problema ms grave, el software no ha podido realizar alguna funcin.

  • CRITICAL (nivel 50): Un error grave, que indica que es posible que el propio programa no pueda seguir ejecutndose.

Qu son los Handlers?

Los handlers (manejadores) del mdulo logging de Python son componentes encargados de dirigir los registros (logs) generados por un Logger a sus destinos finales, como la consola, archivos, o correos electrnicos. Actan como el puente entre el mensaje de registro y el medio de salida, permitiendo filtrar y dar formato a los mensajes.

Aspectos clave de los Handlers:

  • Destinos mltiples: Un mismo logger puede tener mltiples handlers (por ejemplo, registrar errores en un archivo y mostrar avisos en la consola).

  • Filtrado de Nivel: Cada handler puede tener su propio nivel de severidad (DEBUG, INFO, ERROR), filtrando mensajes independientemente del nivel del logger.

  • Formatters: Permiten asignar un formato especfico a la salida de cada manejador.

Tipos comunes:

  • StreamHandler: Enva logs a la consola (stdout/stderr).
  • FileHandler: Guarda los logs en un archivo. RotatingFileHandler: Gestiona archivos de log, rotndolos al alcanzar cierto tamao.
  • SMTPHandler: Enva logs por correo electrnico.

En resumen, los handlers definen dnde y cmo se almacenan o visualizan los mensajes de log en una aplicacin Python.
